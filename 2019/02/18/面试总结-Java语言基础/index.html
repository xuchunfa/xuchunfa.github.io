1.java基本数据类型？

byte(1)、boolean(1)、short(2)、char(2)、int(4)、float(4)、long(8)、double(8)
  1. private修饰的方法可以通过反射访问，那么private的意义何在？

  2. 局部变量使用前需要显式地赋值，否则编译通过不了，为什么这么设计？

    约束限制，减少出错。

  3. 接口和抽象的区别？

    1.抽象类里面可以拥有成员变量和自己的非抽象方法，接口中成员默认为public static final，所有方法都是抽象的。

2.抽象是通过发现子类公有的成员和方法，将其抽出来形成抽象类；而实现接口的子类可能不存在任何联系。即抽象类是自底向上设计的，接口是自顶向下设计的。

3.门和报警器的例子。将门抽象为一个类，实现报警功能的接口。抽象类是一种“is-a”的关系，接口代表的是“like-a”的关系。

  4. 重写和重载的区别？

    重载：

    1.函数名相同，参数列表不同；

2.前期绑定，即编译期绑定，编译器确定调用哪个方法；

    重写：

    1.子类重写父类中的方法，函数名和参数列表都相同;

2.动态绑定，多态

  5. Object类有哪些方法？以及具体用到哪？

     1.getClass()、hashCode()、equals(Object)、clone()、toString()、notify()、notifyAll()、wait(long)、wait(）、finalize();

2.wait()和notify():生产者消费者模式中常用到；

  1. java的浅拷贝和深拷贝？

    浅拷贝：

    1.属性为引用类型，拷贝的是内存地址，如果一个对象改变了引用对象指向的对象的内容，那么拷贝对象中也会发生变化；

2.的确创建了一个对象；

    深拷贝：

    1.重写引用对象的克隆方法，即可实现深拷贝；

class Father implements Cloneable{
    private String fatherName;
    private Child child;

    public Father(String fatherName,Child child) {
        this.fatherName = fatherName;
        this.child = child;
    }

    @Override
    protected Object clone() throws CloneNotSupportedException {
        //重写clone方法
        Father father = (Father) super.clone();
        father.child = (Child) father.child.clone();
        return father;
    }
}

class Child implements Cloneable{
    private String childName;

    public Child(String childName) {
        this.childName = childName;
    }

    @Override
    protected Object clone() throws CloneNotSupportedException {
        return super.clone();
    }
}

@Test
public void test() throws CloneNotSupportedException {
    Child child = new Child("xiaoxu");
    Father father = new Father("daxu",child);
    Father clonedFather = (Father) father.clone();
    System.out.println(father.child == clonedFather.child);//false
}

    总结：

    如果一个对象内部只有基本数据类型，那用 clone() 方法获取到的就是这个对象的深拷贝，

而如果其内部还有引用数据类型，那用 clone() 方法就是一次浅拷贝的操作。

  1. equals和==的区别以及hashcode和equals之间的关系？

    1.==表示地址是否相等，equals表示内容是否相等；

    hashcode和equals之间的关系：

    1.两个对象equals相等，则hashcode相等；

2.两个对象hashcode相等，equals不一定相等；

3.当equals()被重写的时候，hashcode（）也要被重写，保证相等的对象的hashcode相等；

4.java的集合中通常使用equals（）判断对象是否相等，使用hashcode能够加快效率；

8.StringBuffer和StringBuilder的区别？

String常量字符串进行拼接通常是最慢的，而且开销大。

1.StringBuilder适合单线程、速度快、线程不安全；StringBuffer线程安全；

2.一般字符串相加不会有多线程操作，推荐使用StringBuilder;

3.如果要对全局变量字符串进行拼接建议使用StringBuffer；

//JDK1.7 以后字符串常量池也在堆里面了
public class Main {
    public static void main(String[] args) {
        String s1 = "hello";
        String s2 = "hello";
        String s3 = "he" + "llo";
        String s4 = "hel" + new String("lo");
        String s5 = new String("hello");
        String s6 = s5.intern();
        String s7 = "h";

        System.out.println(s1==s2);//true
        System.out.println(s1==s3);//true 编译优化
        System.out.println(s1==s4);//false s1常量池地址 s4普通堆地址
        System.out.println(s1==s9);//false s1常量池地址 s9普通堆地址
        System.out.println(s4==s5);//false s4普通堆地址 s5普通堆地址
        System.out.println(s1==s6);//true s1常量池地址 s6常量池地址
}

String.intern()方法：

使一个位于堆中的字符串在运行期间动态地加入到字符串常量池中，如果字符串常量池中已有该对象
对应的字面量，则直接返回该字面量在字符串常量池中的引用。

8.String的intern()方法？

  1.String 的 String pool是一个固定大小的HashTable，默认大小1009；

2.intern方法（JDK1.6）：如果常量池中(方法区Perm)存在当前字符串, 就会直接返回当前字符串. 如果常量池中没有此字符串, 会将此字符串放入常量池中后, 再返回它的引用；

3.String s = new String("1")创建了两个对象，一个存放在常量池，另一个存放在Heap区；

4.intern重用String对象，节省了内存消耗；

美团技术讲解intern

//JDK1.7情况下（字符串常量池已经移到正常的Heap区）
public void test1(){
    String s1 = new String("1");
    s1.intern();
    String s2 = "1";
    System.out.println(s1 == s2);//false
    String s3 = new String("2") + new String("1");
    s3.intern();//直接存储堆中的引用，即 s3.intern()==s3 为true
    String s4 = "11";
    System.out.println(s3 == s4);//true
}


//JDK1.6情况下(字符串常量池放在perm区)
public void test1(){
    String s1 = new String("1");
    s1.intern();
    String s2 = "1";
    System.out.println(s1 == s2);//false

    String s3 = new String("1") + new String("1");
    s3.intern();
    String s4 = "11";
    System.out.println(s3 == s4);//false
}
