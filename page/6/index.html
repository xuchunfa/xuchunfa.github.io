数组中的重复数字

发表于 2018-07-18 | 分类于 数组 | 阅读次数:

长度为n的数组里面的所有数字都在0~n-1的范围内，找出数组中的重复数字？

核心思想：

1.数组下标 = 数组元素

2.目标交换：num[i] = m

swap: m <---> num[m] 直到 num[i] == i

public class OptDuplication {

//哈希表的方法解决，时间复杂度：O(n) 空间复杂度：O(n)
boolean findDup1(int[] nums,int len,int[] duplication){

    if(nums == null || len <= 0){
        System.out.println("数组必须存在且不能为空!");
        return  false;
    }

    for(int i = 0;i<len;i++){
        if(nums[i]<0 || nums[i]>len-1) {
            System.out.println("数组元素应在0 ~ len-1");
            return false;
        }
    }

    //哈希表
    int[] count = new int[len];
    for(int i = 0;i<len;i++){
        if(count[nums[i]] == 1){
            duplication[0] = nums[i];
            return true;
        }

        count[nums[i]]++;
    }

    return false;

}

boolean findDup(int[] nums,int len,int[] duplication){

    if(nums == null || len <= 0){
        System.out.println("数组必须存在且不能为空!");
        return  false;
    }

    for(int i = 0;i<len;i++){
        if(nums[i]<0 || nums[i]>len-1) {
            System.out.println("数组元素应在0 ~ len-1");
            return false;
        }
    }

    int temp;
    for(int i = 0;i<len;i++){
        while (i != nums[i]){
            //nums[i]=m 说明 m 所在的下标是 i 和 m 所以重复了
            if(nums[i] == nums[nums[i]]){
                duplication[0]=nums[i];
                return true;
            }else {
                //交换num[i] 和 num[num[i]]
                temp = nums[i];
                nums[i] = nums[nums[i]];
                nums[temp] = temp;
            }
        }
    }

    System.out.println("没有重复数字");
    return false;
}

长度为n+1的数组里面的所有数字都在1~n的范围内，所以必定存在重复的数字，找出数组中的重复数字？

核心思想：

1.二分法的运用，加入了统计数字个数的方法

2.循环结束条件：start == end

public class DuplicationVersion {

//时间复杂度:O(nlogn) 空间复杂度:O(1)
boolean findDup(int[] nums,int len,int[] duplication){

    if(nums == null || len <= 0){
        System.out.println("数组必须存在且不能为空!");
        return  false;
    }

    for(int i = 0;i<len;i++){
        if(nums[i]<=0 || nums[i]>len-1) {
            System.out.println("数组元素应在1 ~ "+ (len-1)+ "之间");
            return false;
        }
    }

    int start = 1;
    int end = len-1;
    int middle;
    while(start <=  end){

        middle = (start + end)/2;
        int number = calNum(nums,len,start,middle);

        //结束条件
        if(start == end){
            if(number > 1){
                duplication[0] = start;
                return true;
            }else{
                break;
            }
        }

        if(middle - start + 1 < number){
            end = middle;
        }else {
            start = middle + 1;
        }
    }

    System.out.println("没有重复数字");
    return false;
}

private int calNum(int[] nums, int len, int start, int end) {
    int count = 0;
    for(int i = 0;i<len;i++){
        if(nums[i] >= start && nums[i] <= end){
            count++;
        }
    }
    return count;
}
 }

单例模式

发表于 2018-07-17 | 分类于 Java设计模式 | 阅读次数:

  • 单例模式指？

    只生成一个对象，避免重复创建对象。

  • 单例模式中构造函数为什么私有化？

    1.构造函数私有化后，那么只能通过这个类的内部函数才能构造这个类的对象。

    2.将生成类对象的方法用static修饰的原因是：因为静态方法不需要依赖实例化的对象。

  • 单例模式中volatile变量的作用？

    首先分析 instance = new Singleton() 的本质：

    1.为对象分配内存空间

    2.对象初始化

    3.将对象的引用指向对应的内存地址。

但是在单线程内2和3可能会发生重排序，于是发生了如下情况：

此时线程B访问的是没有经过初始化的对象，会报错，所以使用volatile的根源在于：禁止A2和A3的重排序。

  • 单例模式中为什么要使用双重检查索(DCL)？

    因为在多线程的环境下，所有线程在调用getSingleton()方法时都必须在synchronized这里排队，而加锁是一个非常耗时的操作，这导致单例模式的效率非常低。我们只需要在实例还没有创建之前添加加锁操作。

  • 线程安全的延迟初始化？

    1.基于volatile

    2.基于类初始化

    对于2举个例子：

    private static class Holder{

    private static Singleton instance =  new Singleton();

    }

    public static Singleton getInstance(int i){

    return Holder.instance;

    }

  • 静态内部类方法创建单例模式的好处？

    按需创建实例（饿汉式模式：在类加载时就完成了初始化），提高了内存的利用效率。

    这种方式的特点是：Singleton类被装载了，instance不一定被初始化，只有显示通过调用getInstance方法时，才会显示装载SingletonHolder类，从而实例化instance。

  • 栈内存和堆内存的比较？

**堆内存存的内容：**数组元素、类对象、类中的非静态成员变量（实例变量）、包装类数据(Integer等)、

**栈内存内容：**

1.方法中声明的变量(因为方法执行时，被分配的内存就在栈中,即方法栈)：分基本数据类型变量和引用变量，前者的变量名和值都存在方法栈中，后者声明的变量存放在栈中，所指的对象放在堆中。

2.对象的引用

**方法区：**线程共享的内存区域。类里面的静态成员和静态方法、

**常量池：**方法区的一块内存。

**静态域：**方法区的一块内存。存放类中的静态成员变量

  • 局部变量保存在栈内存中，随方法的消失而消失;成员变量存储在堆中的对象里，由垃圾回收器负责回收。

  • 包装类数据Integer、String等等存在于堆中。

  • Stirng s2 = new String(“xuchunfa”) 这个对象存放在堆里面。具体过程：
    先去常量池去找，看有没有"xuchunfa"对象，如果没有则在常量池里面创建一个对象，如果有的话则在堆里面再创建"xuchunfa"对象的拷贝对象

  • 即equls()相同的字符串，在常量池中只有一份，在堆中可以有多份

  • String s1 = “xuchunfa”的理解：

    不能肯定的说创建了一个"xuchunfa"的对象，因为常量池中可能已经存在。只能肯定的说创建了指向"xuchunfa"对象的引用变量。new() 方法才是显示地创建一个对象

  • 当字符串在堆中被分配内容时，它就是不可变的，任何String的方法无法改变字符串本身。但是它创建的引用变量可以指向新的String对象。如图：

  • 字符串不可变的原因？

    String s1 = "as"; String s2 = "as"两句话创建了一个"as"对象，两个引用。如果s1指向的"as"对象发生了变化，那么s2原本指向的对象就会找不到了。

字符串转整型

发表于 2018-07-12 | 分类于 字符串 | 阅读次数:

需要考虑的临界条件

  • Integer的范围-2的31次方 到 +2的31次方减1
  • 整型可以从最小2进制到最大36进制
  • 字符串为空时抛异常
  • 字符串有非数字字符时抛异常
  • 字符串有正负号
  • 字符串就只有正负号，即长度为1抛出异常

遇到的问题

  • 怎么判断第一个字符是非数字符？

    就只考虑firstChar < '0' 的情况，如果符合条件，说明一定有正负号，否则抛非法字符的异常，i++ 如果firstChar > '0' 那么while循环从第一个开始(i=0)

  • 怎么实现’1’-> 1 的转换？

    ch-'0'实现

  • 怎么设置limit?

    错误：

    正号时：limit = Integer.MAX_VALUE；

    负号时：limit = -Integer.MIN_VALUE；

    条件：result*10 + digit > limit

    计算：result = result*10 + digit

    limit的类型为int,越界了！！！

正确：

正号时：limit = -Integer.MAX_VALUE；

负号时：limit = Integer.MIN_VALUE；

条件：result*10 - digit < limit 为true时 越界

计算：result = result*10 - digit

  • 容易忽略的地方？

    例如：str = “21474836471”

    方法：

    int flag = limit/radix;

    if(result < flag){

    越界

    }

    result = result*radix;

public class StringToInteger {

public static int vary(String s) {

    int i = 0;
    int len = s.length();
    int result = 0;
    int radix = 10;
    boolean negative = false;//默认正数
    int limit = -Integer.MAX_VALUE;//-2147483647

   /* if(s == null){
        System.out.println("null");
        return -1;
    }*/

    if (s == null || s.length() == 0) {
        System.out.println("字符串不能为空!");
        return -1;
    }

    if (len > 0) {
        char firstChar = s.charAt(i);

        //判断第一个字符是不是正负号

            if (firstChar < '0') {
            if (firstChar == '-') {
                limit = Integer.MIN_VALUE;//-2147483648
                negative = true;
            } else if (firstChar != '+') {
                System.out.println("非法字符!");
                return -1;
            }

            if (len == 1) {
                System.out.println("非法格式");
                return -1;
            }
            i++;
        }

        //用于判断result < Integer.MIN_VALUE 的情况!!!
        int flag = limit/radix;


        //接着开始循环
        while (i < len) {
            char ch = s.charAt(i);
            if (ch >= '0' && ch <= '9') {
                int digit = ch - '0';

                if(result < flag){
                    System.out.println("越界!");
                    return -1;
                }
                //这里result定义的为int型,有可能越界
                result = result * radix;
                if (result - digit < limit) {
                    System.out.println("越界!");
                    return -1;
                }
                result -= digit;

            } else {
                System.out.println("非法字符!");
                return -1;
            }
            i++;
        }
    }
    //结束 if(len > 0)
    return negative ? result : -result;
}

public static void main(String[] args) {
    String s = "";
    //String s = "";
    //String s = "123aaa";
    //String s = "2147483647";
    //String s = "2147483648";
    //String s = "2147483641222";
    //String s = "21474";
    //String s = "-1";
    //String s = "-2147483648";
    //String s = "-2147483649";
    //String s = "-2147483641";

    System.out.println(StringToInteger.vary(s));
}
}

Github常用命令总结

发表于 2018-07-04 | 分类于 Github | 阅读次数:

  • master: 本地项目
  • origin: Fork的远程项目
  • upstream: 别人的开源项目

git pull = git fetch + git merge

例如：
git pull origin next
等价于

` git fetch origin`
 `git merge origin/next`

常用命令图解：

  • 代码提交

    例如：提交工作区自上次commit之后的变化，直接到仓库区

    git commit -a

    例如：提交暂存区到仓库区

    git commit -m [message]

  • 切换分支

    例如：切换到指定分支，并更新工作区

    git checkout [branch-name]

  • 合并分支

    例如：合并指定分支到当前分支

    git merge [branch]

  • 查看远程仓库

    `git remote -v `

  • 添加远程仓库

    `git remote add pb https://github.com/paulboone/ticgit`

  • 从远程仓库中抓取与拉取

    git fetch [remote-name]

  • 推送到远程仓库

    git push [remote-name] [branch-name]

  • 在origin/master基础上创建一个新分支

    git checkout -b newBranch origin/master

  • 在本地分支上合并远程分支

    git merge origin/master

    或者

    git rebase origin/master

  • 取回远程主机某个分支的更新，再与本地的指定分支合并

    git pull <远程主机名> <远程分支名>:<本地分支名>

    例如：

    `git pull origin next:master`

    远程分支与当前分支合并：

    `git pull origin next`

  • 将本地分支更新，推送到远程主机

    git push <远程主机名> <本地分支名>:<远程分支名>

    不要弄混淆了：

    pull 拉取远程分支名

push 推送本地分支名

    例如：将本地的branch1分支推送到origin主机的master分支

    `git push origin branch1`
