移位实现乘除运算

除2 = 右移1位 乘2 = 左移1位

除4 = 右移2位 乘4 = 左移2位

除8 = 右移3位 乘8 = 左移3位

取余运算用位运算与来代替

A % B - A & (B - 1)其中满足 A>0 And B = 2^n

其中>>>表示：无符号右移，忽略符号位，空位都以0补齐,在hashMap中有用到。

要注意带符号类型的左右移位

将负值向右移位可生成原始值一半的值（向下舍入）。 例如，-253（二进制 11111111 00000011）向右移动 1 位会生成 -127（二进制 11111111 10000001）。 将 + 253 向右移位生成 +126。

注：

1.如果数字是有符号正数，右移n位，则在最左边补n个0;

2.如果数字是有符号负数，右移n位，则在最左边补n个1;

3.如果一个数字，左移n位，则在最右边补n个0;

一个数减去一后再与原本的自己做位与运算，得到的结果是将原数二进制中最右边的1变为0

动态规划问题的特点

  • 大问题可以分解成小问题，每个小问题都存在最优解。

  • 整体问题最优解依赖子问题的最优解。

  • 小问题之间还会存在相互重叠的更小的问题。

  • 从上往下分析问题，从下往上求解问题。

剪绳子把长度为 n 的绳子剪成 m 段(n > 1,m > 1)所有相乘乘积最小？

public class CutStrip {

public static int cutStrip(int length){
    if(length < 2)
        return 0;

    if(length == 2)
        return 1;
    if(length == 3)
        return 2;

    int[] result = new int[length+1];
    result[1] = 1;
    result[2] = 2;
    result[3] = 3;

    for (int i = 4;i<=length;i++){
        int max = 0;
        for(int j = 1;j<=i/2;j++){
            int temp = result[j]*result[i-j];
            if(temp > max)
                max = temp;
        }
        result[i] = max;
    }

    return result[length];
}

public static void main(String[] args){
    System.out.println(CutStrip.cutStrip(10));
}
}

回溯和递归的理解？

1.如果一个问题的子问题和自身类似，只是规模和条件上发生变化，那么一套代码就可以解决二者，这就是递归。

2.如果一个问题的解存在于一个可以深度遍历的解空间中，通过深度优先遍历找到解的过程就是回溯 。回溯有的时候可以用递归解决。

3.回溯法可以通过递归方法列出所有的解空间。解空间还可以用迭代的方法实现。

    public class MatrixPath {

public static boolean hasPath(char[][] a,int rows,int cols,String str){
    if(a == null || rows < 1 || cols < 1 || str == null)
        throw new RuntimeException("参数非法");

    boolean[][] visited = new boolean[rows][cols];
    int pathLength = 0;
    for(int row = 0;row < rows;row++){
        for(int col = 0;col < cols;col++){
            if(hasOnePath(a,row,col,rows,cols,str,pathLength,visited)){
                return true;
        }
    }
}

    return false;
}

private static boolean hasOnePath(char[][] a, int row, int col, int rows, int cols, String str,int pathLength,boolean[][] visited) {
    boolean result = false;
    if(str.length() == pathLength){
        return true;
    }

    if(row >= 0 && row < rows && col >= 0 && col < cols && a[row][col] == str.charAt(pathLength) && !visited[row][col]) {
        ++pathLength;
        visited[row][col] = true;

        result = hasOnePath(a, row - 1, col, rows, cols, str, pathLength, visited) ||
            hasOnePath(a, row, col - 1, rows, cols, str, pathLength, visited) ||
            hasOnePath(a, row + 1, col, rows, cols, str, pathLength, visited) ||
            hasOnePath(a, row, col + 1, rows, cols, str, pathLength, visited);

        if (!result) {
            --pathLength;
            visited[row][col] = false;
        }
    }

    return result;
}

为什么要使用线程池？

1.降低资源消耗。重复利用已经创建的线程。因为线程的创建和销毁是需要消耗java虚拟机内存的。

2.提高响应速度。任务不需要等待线程的创建(当然是在线程池中的线程还没完全被消耗完的情况下)

3.提高线程的可管理性。

线程池执行流程？

线程池中线程创建的时机？

1.判断核心线程池是否已满，否则提交任务的时候创建新的线程，并由这个线程执行这个任务，是则进入下步；

2.判断工作队列是否已满，否的话任务存储在该工作队列，等待线程池中的线程通过while循环从队列里面
拉取任务，是则进入下步；

3.判断线程池是否已满，否的话就创建新线程执行任务，并由这个线程执行这个任务，是则进入下步；

4.交给饱和策略处理；

任务在执行的过程中发生了异常怎么处理？

某个任务执行时出现异常，那么执行任务的线程将会关闭（移除），然后启动一个新的线程来代替它，
新的线程也会不断的从队列里面获取任务执行。

解决办法：

在执行业务代码时，自己try...catch捕获异常。

看一下源码理解：

private void processWorkerExit(Worker w, boolean completedAbruptly) {
    if (completedAbruptly) //执行任务时出现异常
        decrementWorkerCount();

    final ReentrantLock mainLock = this.mainLock;
    mainLock.lock();
    try {
        completedTaskCount += w.completedTasks;
        workers.remove(w);//移除出现异常的线程
    } finally {
        mainLock.unlock();
    }

    tryTerminate();

    int c = ctl.get();
    if (runStateLessThan(c, STOP)) {
        if (!completedAbruptly) {
            int min = allowCoreThreadTimeOut ? 0 : corePoolSize;
            if (min == 0 && ! workQueue.isEmpty())
                min = 1;
            if (workerCountOf(c) >= min)//例如如果有空闲线程，就不用创建新的线程来代替
                return; // replacement not needed
        }
        addWorker(null, false);//创建新的线程代替出现异常的线程
    }
}

Worker加锁的原因？

因为外部有可能在线程池里面调用了其他方法中断Worker,为了保证任务完整被执行，必须在获得锁的情况下
才能中断Worker.

1.corePoolSize:线程池中线程的基本数量。（工人）

2.maxinumPoolsize：任务队列满了之后，满足条件：当前线程数 < maxinumPoolsize 又会创建线程。且：
当前线程数 + 新创建的线程数 <= maxinumPoolsize（工人+临时工）

3.workerQueue:用来存放任务的队列。是通过调用方法execute()来实现的。

类型：BlockingQueue<Runnable>;

4.keepAliveTime:工作线程空闲时的存活时间。默认情况下，只有当线程池中的线程数大于corePoolSize时，keepAliveTime才会起作用。如果设置allowCoreThreadTimeOut为true那么这个参数对所有线程有效；

5.handler：表示当拒绝处理任务时的策略。

6.poolSize：线程池中当前的线程数。

拒绝策略(队列和线程池满了)：

1.AbortPolicy:默认策略，直接抛出异常RejectedExecutionException；

2.DiscardPolicy:不处理直接抛弃；

3.DiscardOldestPolicy:如果线程池没有关闭，丢弃队列最前面的任务，执行当前任务；

4.CallerRunsPolicy:如果线程池没有关闭，由调用线程（例如main线程）执行任务；

创建线程池的方式：(Executors框架提供)

1.newFixedThreadPool：创建固定数量的线程池。默认使用的是无界队列，maximumPoolSize和keepAliveTime参数无效，不会拒绝任务。
适合于负载较重的服务器。

ThreadPoolExecutor源码解析：

1.将Integer32位分成两部分，高3位表示线程池的状态，低29位表示线程的最大数量；

2.线程池的状态：

    2.1 RUNNING:接收新的任务，处理等待队列中的任务；

    2.2 SHUTDOWN:不接收新的任务，但会处理等待队列中的任务（调用shutdown方法）；

    2.3 STOP:不接收新的任务,不处理等待队列中的任务，interrupt正在执行任务的线程（调用shutdownNow方法）；

    2.4 TIDYING:所有的任务都销毁了，线程池被清空；

    2.5 TERMINATED：terminated()方法执行完成后的状态；

3.状态大小的比较

    RUNNING < SHUTDOWN < STOP < TIDYING < TERMINATED

//ThreadPoolExecutor.java
public void execute(Runnable command) {
    if (command == null)
        throw new NullPointerException();

    int c = ctl.get();
    if (workerCountOf(c) < corePoolSize) {//当前线程数小于核心线程数
        if (addWorker(command, true))
            return;
        c = ctl.get();
    }

    //线程数大于corePoolSize或者addWorker失败，会执行到这
    /////isRunning的源码

        private static boolean isRunning(int c) {
            return c < SHUTDOWN;
        }
    /////

    if (isRunning(c) && workQueue.offer(command)) {
        int recheck = ctl.get();

        if (! isRunning(recheck) && remove(command))
            reject(command)；//线程池不处于Running状态，移除已经入队的任务
        else if (workerCountOf(recheck) == 0)
            //任务提交到了队列，但线程数为0，开启新的线程
            addWorker(null, false);
    }

    //执行到这，说明任务队列已满，以maximumPoolSize为界创建线程
    else if (!addWorker(command, false))
        reject(command);//线程数大于maximumPoolSize时执行任务拒绝策略
}

创建线程（addWorker）:

private boolean addWorker(Runnable firstTask, boolean core) {
    retry:
    for (;;) {
        int c = ctl.get();
        int rs = runStateOf(c);

          //当线程池状态为SHUTDOWM、STOP、TIDYING、TERMINATED时
        //且满足：
            1.rs != SHUTDOWN
            2.firstTask != null
            3.workQueue.isEmpty()队列为空

        //主要就是当线程池关闭时，提交任务会失败
        //rs == SHUTDOWN && firstTask = null &! workQueue.isEmpty()仍然成功
        if (rs >= SHUTDOWN &&
            ! (rs == SHUTDOWN &&
               firstTask == null &&
               ! workQueue.isEmpty()))
            return false;

        for (;;) {
            int wc = workerCountOf(c);
            if (wc >= CAPACITY ||
                wc >= (core ? corePoolSize : maximumPoolSize))
                return false;
            if (compareAndIncrementWorkerCount(c))
                break retry;//条件校验工作完毕，准备创建线程

            //执行到这说明CAS失败，有其他线程创建完毕
            c = ctl.get();
            if (runStateOf(c) != rs)
                continue retry;//状态变更需要回到最外层的for循环
        }
    }

    //开始创建线程
    boolean workerStarted = false;
    boolean workerAdded = false;
    Worker w = null;
    try {
        w = new Worker(firstTask);
        final Thread t = w.thread;
        if (t != null) {
            final ReentrantLock mainLock = this.mainLock;
            mainLock.lock();
            try {

                int rs = runStateOf(ctl.get());
                //进入if的条件
                    1.线程池处于RUUNING状态；
                    2.线程池处于SHUTDOWN状态，且未提交任务；

                if (rs < SHUTDOWN ||
                    (rs == SHUTDOWN && firstTask == null)) {
                    if (t.isAlive()) 
                        throw new IllegalThreadStateException();
                    workers.add(w);
                    int s = workers.size();
                    if (s > largestPoolSize)//记录线程池线程曾经达到的最大数量
                        largestPoolSize = s;
                    workerAdded = true;
                }
            } finally {
                mainLock.unlock();
            }
            if (workerAdded) {
                t.start();//启动线程，真正执行的是Worker中的run方法
                workerStarted = true;
            }
        }
    } finally {
        if (! workerStarted)
            addWorkerFailed(w);//失败后的处理，移除线程以及workCount的还原
    }
    return workerStarted;//返回添加线程是否成功的布尔值
}

启动线程时调用Worker的run方法：

public void run() {
    runWorker(this);
}

final void runWorker(Worker w) {
    Thread wt = Thread.currentThread();
    Runnable task = w.firstTask;
    w.firstTask = null;
    w.unlock(); // allow interrupts
    boolean completedAbruptly = true;
    try {
        //执行指定的firstTask或者从任务队列中取任务执行
        //while循环一直获取任务执行
        while (task != null || (task = getTask()) != null) {
            w.lock();
            //STOP状态中断线程
            if ((runStateAtLeast(ctl.get(), STOP) ||
                 (Thread.interrupted() &&
                  runStateAtLeast(ctl.get(), STOP))) &&
                !wt.isInterrupted())
                wt.interrupt();
            try {
                beforeExecute(wt, task);
                Throwable thrown = null;
                try {
                    task.run();//真正执行任务
                } catch (RuntimeException x) {
                    thrown = x; throw x;
                } catch (Error x) {
                    thrown = x; throw x;
                } catch (Throwable x) {
                    thrown = x; throw new Error(x);
                } finally {
                    afterExecute(task, thrown);
                }
            } finally {
                task = null;
                w.completedTasks++;
                w.unlock();
            }
        }
        completedAbruptly = false;
    } finally {
        processWorkerExit(w, completedAbruptly);
    }
}

看一下获取队列任务的源码：

private Runnable getTask() {
    boolean timedOut = false; // Did the last poll() time out?

    for (;;) {
        int c = ctl.get();
        int rs = runStateOf(c);

        //两种情况：
            1.rs >= STOP
            2.rs = SHUTDOWN && workQueue.isEmpty()
            补充：
                上面说到过，如果rs = SHUTDOWN && ！workQueue.isEmpty()仍然会
            执行完队列中的任务；
        if (rs >= SHUTDOWN && (rs >= STOP || workQueue.isEmpty())) {
            decrementWorkerCount();
            return null;
        }

        int wc = workerCountOf(c);
        boolean timed = allowCoreThreadTimeOut || wc > corePoolSize;

        if ((wc > maximumPoolSize || (timed && timedOut))
            && (wc > 1 || workQueue.isEmpty())) {
            if (compareAndDecrementWorkerCount(c))
                return null;
            continue;
        }

        //wc <= maximumPoolSize && 没有超时
        try {
            Runnable r = timed ?
                workQueue.poll(keepAliveTime, TimeUnit.NANOSECONDS) :
                workQueue.take();//一旦列队为空，线程就会发生阻塞
            if (r != null)
                return r;//返回任务
            timedOut = true;
        } catch (InterruptedException retry) {
            timedOut = false;
        }
    }
}

桶的思想？

public class RadixSort {

public static void radixSort(int[] a,int k){
    int[] counter = new int[10];
    int[][] container = new int[10][a.length];
    int n = 1;
    int t = 0;
    while (n < k){
        for(int num : a){
            int radix = (num/n)%10;
            container[radix][counter[radix]] = num;
            counter[radix]++;
        }

        for(int i = 0;i < 10;i++){
                if(counter[i] != 0){
                    for(int j = 0;j<counter[i];j++){
                        a[t] = container[i][j];
                        t++;
                    }
                }
            counter[i] = 0;
        }

        t = 0;
        n *= 10;
    }
}

public static void main(String[] args){
    int[] a = {1,21,32,45,687,89,43,44,95};
    RadixSort.radixSort(a,1000);
    System.out.println(Arrays.toString(a));
}
}

发表于 2018-08-04 | 分类于 排序 | 阅读次数:

核心思想：哈希表的思想，确定元素在整个数组的排第几来确定它在最终排序数组中的位置。

计数排序示意图：

public class CountSort {

//数组数字在0 ~ k之间
public static int[] countSort(int[] a,int k){

    int[] elementCounter = new int[k+1];

    for(int i = 0;i<a.length;i++){
        elementCounter[a[i]]++;
    }

    //构造计数数组(即小于等于该元素的个数)
    for(int j = 1;j<elementCounter.length;j++){
        elementCounter[j] = elementCounter[j-1]+elementCounter[j];
    }


    int[] result = new int[a.length];

    //从原数组末尾开始遍历
    for(int i = a.length-1;i>=0;i--){
        result[elementCounter[a[i]]-1] = a[i];
        elementCounter[a[i]]--;
    }

    return result;
}

//优化方法只用了一个辅助数组
public static void optCountSort(int[] a,int k){
    int[] c = new int[k+1];
    for(int i = 0;i < a.length;i++){
        c[a[i]]++;
    }
    int z = 0;
    for(int j = 0;j <= k;j++){
        while (c[j] > 0){
            a[z++] = j;
            c[j]--;
        }
    }
}

public static void main(String[] args){
    int[] a = {6,5,4,3,2,1};
    int[] result = new int[a.length];
    result = CountSort.countSort(a,6);
    System.out.println(Arrays.toString(result));
}
}

public class ShellSort {

public static void shellSort(int[] a){
    for(int increment = a.length/2;increment > 0 ;increment /= 2){
        for(int i = increment;i<a.length;i++){
            int j = i;
            while (j - increment >=0  && a[j-increment] > a[j]){
                //交换排序
                swap(a,j,j-increment);
                j -= increment;
        }
        }
    }
}

private static void swap(int[] a, int i, int j) {
    int temp = a[i];
    a[i] = a[j];
    a[j] = temp;
}

public static void shellSort1(int[] a){
    for(int increment = a.length/2;increment > 0 ;increment /= 2){
        for(int i = increment;i<a.length;i++){
            int j = i;
            int temp = a[j];

            //直接插入
            while(j-increment >= 0 && a[j-increment] > temp){
                a[j] = a[j-increment];
                j -= increment;
            }
            a[j] = temp;
        }
    }
}

public static void main(String[] args){
    int[] a = {9,8,7,6,5,4,3,2,1};
    ShellSort.shellSort1(a);
    System.out.println(Arrays.toString(a));
}
}

归并排序

发表于 2018-08-02 | 分类于 排序 | 阅读次数:

什么叫分治思想？

分治法将问题分(divide)成一些小的问题然后递归求解，而治(conquer)的阶段则将分的阶段得到的各答案”修补”在一起，即分而治之

public class MergeSort {

//临时数组
public static void sort(int[] a){
    int[] temp = new int[a.length];
    mergeSort(a,0,a.length-1,temp);
}

//治的思想
public static void merge(int[] a,int left,int mid,int right,int[] temp){
    int i = left;
    int j = mid + 1;
    int k = 0;
    while(i <= mid && j <= right){
        if(a[i] < a[j]){
            temp[k++] = a[i++];
        }else{
            temp[k++] = a[j++];
        }
    }

    while (i <= mid){
        temp[k++] = a[i++];
    }

    while (j <= right){
        temp[k++] = a[j++];
    }

    k = 0;
    while (left <= right){
        a[left++] = temp[k++];
    }
}

//归并排序
public static void mergeSort(int[] a,int left,int right,int[] temp){

    if(a == null){
        throw new RuntimeException("输入数组");
    }

    if(left < 0 || left >= a.length || right < 0 || right >= a.length ){
        throw new RuntimeException("越界");
    }

    if(left >= right)
        return;

    int mid = (left + right)/2;

    //分
    mergeSort(a,left,mid,temp);
    mergeSort(a,mid+1,right,temp);

    //治
    merge(a,left,mid,right,temp);
}

public static void main(String[] args){
    int[] a = {6,5,4,4,4,3,2,1};
    sort(a);
    System.out.println(Arrays.toString(a));
}
}

进程和线程分析

发表于 2018-08-02 | 分类于 Java并发编程 | 阅读次数:

线程中Blocked 和 Waiting 状态的区别？

Blocked:线程在等待监视器锁(例如synchronized方法或者块)或者线程在调用过对象的wait()方法后，重新被唤醒时(例如：notify()),此时会进入Blocked状态,试图获得调用对象的锁。

Waiting:
由于主动调用了以下方法而进入的状态：
1.Object.wait()
2.Thread.join()
3.LockSupport.park()

Time_waiting:
调用了以下方法：
1.Thread.sleep
2.Object.wait(long)
3.Thread.join(long)
4.LockSupport.parkNanos
5.LockSupport.parkUntil

同步队列和等待队列的区别？

同步队列：存放竞争同步资源的线程的引用。

等待队列：存放着等待被唤醒的线程的引用。

等待/通知的经典范式如下：
等待方
synchronized(对象){
    while(条件不满足){
        对象.wait()
    }
    处理逻辑
}

通知方
synchronized(对象){
    改变条件
    对象.notifyAll()
}

等待超时模式如下：
public synchronized Object get(long mills) throw InterruptedException{
if(mills <= 0){
    while(result == null){
        wait();
    }
     return result;
}else{
    long future = System.currentTimeMillis() + mills;
    long remaining = mills;
    while(result == null && remaining > 0){
        wait(remaining);
        remaining = future - System.currentTimeMillis();
    }
    result == null;
    if(result != null){
        return result;
    }
    return result;
}

进程和线程的区别？

进程：资源分配的最小单位。可以看成是每个运行中的程序。它可以拥有自己独立的资源。有自己的内存空间。

线程：操作系统调度的最小单元。线程是进程的一个实体，这些线程共同享有进程占有的资源和内存空间。

进程和线程的联系：一个进程里面可以创建多个线程，每个线程都拥有各自的计数器、堆栈和局部变量等。

进程和线程的区别：

1.进程之间是相互独立的，不共享内存和数据，线程之间的内存和数据是公用的。

2.一个进程内可以创建多个线程，所有这些线程都共享创建这个进程时所占用的系统资源。

public class HeapSort {

public static void heapSort(int[] a){

    if(a == null){
        throw new RuntimeException("输入数组");
    }

    int len = a.length;
    for(int i = len/2-1;i>=0;i--){
        createHeap(a,i,len);
    }

    for(int j = a.length-1;j>0;j--){
        swap(a,0,j);
        createHeap(a,0,j);
    }
}

private static void swap(int[] a, int i, int j) {
    int temp = a[i];
    a[i] = a[j];
    a[j] = temp;
}

//构建大顶堆
private static void createHeap(int[] a,int i,int len) {

    int father = a[i];
    int j = 2i+1;//j代表子结点坐标 i代表父结点坐标

    while(j < len){

        if(j + 1 < len && a[j] < a[j+1] )
            j++;

        if(a[j] > father){
            a[i] = a[j];
            i = j;
        }else {
            break;
        }
        j = 2*i+1;
    }

    //找到最终位置
    a[i] = father;
}

//构建小顶堆
public static void createMinHeap(int[] a,int i,int length){
    int father = a[i];
    int j = 2*i + 1;
    while (j < length){
        if(j+1 < length && a[j] > a[j+1]){
            j++;
        }
        if(a[j] < father){
            a[i] = a[j];
            i = j;
        }else {
            break;
        }
        j = 2*i+1;
    }
    a[i] = father;
}

public static void main(String[] args){
    int[] a = {1,1,1};
    HeapSort.heapSort(a);
    System.out.println(Arrays.toString(a));
}
}
