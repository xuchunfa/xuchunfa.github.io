排序算法比较

发表于 2018-07-28 | 分类于 算法 ， 排序 | | 阅读次数:

学习过程中一点点积累补充

什么叫做稳定性？

稳定性是指：表示两个值相同的元素在排序前后是否有位置变化。如果前后位置变化，则排序算法是不稳定的，否则是稳定的。

  • 快排分析

    时间复杂度：

    1.最优情况下：两边元素被划分的很均匀，递归次数等价于一颗二叉树的深度logn+1，第一次partition的时候需要和pivot比较n次

    时间复杂度：nlogn

    2.最差情况下：划分为了一颗斜着的树，递归次数为n-1,第一次划分还是得比较n次

    时间复杂度：n^2

    3.一般情况下：略

    时间复杂度：nlogn

    空间复杂度：

    1.最优情况：因为递归调用会消耗空间，空间复杂度取决于栈的调用层数。

    空间复杂度：O(logn)

    2.最差情况：略

    空间复杂度：O(n)

    补充：快排是不稳定的。

  • 直接插入排序

    时间复杂度：

    1.最优情况：正序时，比较次数O(n),移动次数0次。

    时间复杂度：O(n)

    2.最坏情况：反序时，移动次数O(n^2),比较次数也是O(n^2)的。

    时间复杂度：O(n^2)

    平均复杂度:O(n^2)

    空间复杂度：

    空间复杂度：O(1)

    补充：插入排序是稳定的。适用于有一定顺序的数据。

  • 冒泡排序

    时间复杂度：

    1.最优情况：优化后的冒泡排序，只进行了一轮比较，O(n-1)。

    时间复杂度：O(n)

    2.最坏情况：逆序，交换次数O(n^2)。

    时间复杂度:O(n^2)

    平均时间复杂度：O(n^2)

    空间复杂度：O(1)

  • 简单选择排序

    时间复杂度：

    1.最优情况：有序，交换次数0次，比较次数O(n^2)

    时间复杂度：O(n^2)

    2.最坏情况：逆序，交换次数O(n^2)，比较次数O(n^2)

    时间复杂度:O(n^2)

    平均时间复杂度：O(n^2)

    空间复杂度：O(1)

  • 堆排序

    时间复杂度：

    分析：

    初始化堆的过程时间：O(n)

    更改元素后重建时间：O(nlogn) = (n-1)logn = 循环次数*树的深度

    最好 = 最坏 = 平均时间复杂度 = O(nlogn)

    空间复杂度：O(1)

  • 归并排序

    总时间 = 划分时间 + 合并时间

    时间复杂度：

    最好 = 最坏 = 平均 = O(nlogn)

    空间复杂度：

    O(n) = O(logn) + O(n)

  • 希尔排序

    时间复杂度：

    最坏：O(n^2)

    最好：O(nlogn)

    平均：O(n^1.5)

    空间复杂度：O(1)

  • 计数排序

    时间复杂度：

    O(n+K) = 遍历原数组 + 遍历计数数组 + 元素插入新数组

    = O(n) + O(k) + O(n)

    空间复杂度：O(n+k)
